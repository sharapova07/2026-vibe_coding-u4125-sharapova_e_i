# Ally For Projects — Telegram-бот

Бот на **Python** и **python-telegram-bot**: расписание встреч, роли команды, напоминания за 15 минут до начала (часовой пояс **Europe/Moscow**). Состав команды и почты задаются в файле **`team.csv`** (не в коде).

## Требования

- Python 3.9+ (рекомендуется 3.10+)
- Токен бота от [@BotFather](https://t.me/BotFather)
- Файл **`team.csv`** в корне проекта (или путь через `TEAM_CSV_PATH`)

## Установка

```bash
cd ally-for-projects-bot
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Конфигурация

1. Скопируйте пример переменных окружения:

   ```bash
   cp .env.example .env
   ```

2. Откройте `.env` и укажите `BOT_TOKEN`.

3. При необходимости задайте путь к CSV команды (по умолчанию `team.csv` рядом с `bot.py`):

   ```bash
   # TEAM_CSV_PATH=/полный/путь/к/team.csv
   ```

## Файл team.csv

- Кодировка **UTF-8** (поддерживается BOM в начале файла).
- Первая строка — заголовок: `full_name,role,email`
- Каждая следующая строка — один участник; все три поля обязательны.
- Роли **«Руководитель проекта»** и **«Администратор проекта»** должны быть написаны **точно** так же, как в CSV: им соответствуют права на `/add_event`, `/change_event`, `/del_event`, `/list_events`.

При старте бот проверяет наличие и формат `team.csv`. Если файл отсутствует или повреждён, процесс завершится с сообщением в логах.

## Запуск

```bash
source .venv/bin/activate
python bot.py
```

Данные событий и авторизаций хранятся в SQLite (файл `data/bot.sqlite3` по умолчанию). Каталог `data/` создаётся автоматически.

## Команды (кратко)

| Команда | Кто |
|--------|-----|
| `/start` | Все: приветствие и авторизация по ФИО (ФИО должно совпадать с `full_name` в `team.csv`) |
| `/commands` | Авторизованные: список команд |
| `/schedule` | Авторизованные: сегодня / завтра / неделя |
| `/team` | Авторизованные: список команды по ФИО; по кнопке — ФИО, роль и почта |
| `/event <id>` | Авторизованные: карточка события |
| `/add_event`, `/change_event`, `/del_event`, `/list_events` | Только руководитель и администратор проекта (по роли из `team.csv`) |

### Создание мероприятия (`/add_event`)

На шаге 4 кнопки строятся из **`team.csv`**: на каждой — **сокращённое ФИО** (например, `Иванов И. П.`) **и роль в скобках**; отметьте участников **без повторов** и нажмите «Готово — сохранить выбор» (только выбор из списка). В базу по-прежнему сохраняется **полное ФИО** из файла. Если CSV временно недоступен, шаг 4 выполняется **вводом текста** (формат строк: `ФИО — Роль`).

При редактировании (`/change_event`) шаг участников устроен так же; текущий состав события по возможности **предвыбран** в кнопках. Доступна кнопка «Оставить участников без изменений».

## Примечания

- Напоминания получают участники события, у которых есть **совпадающее ФИО** в базе авторизованных пользователей (после `/start`).
- После перезапуска бота отложенные напоминания подхватываются из базы заново.
- Изменения в `team.csv` подхватываются при **следующем чтении файла** (например, при новой команде или шаге диалога). Чтобы гарантированно подтянуть состав после правок файла, достаточно перезапустить бота.
